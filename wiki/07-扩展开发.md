| 扩展点 | 接口 | 注册方式 | 影响范围 |

## 7.2 扩展：新增声学特征提取器

实现 `IAcousticFeatureExtractor` 接口并注册到 DI 容器即可替换或新增声学特征提取逻辑。

### 7.2.1 实现接口

```csharp
// src/FindMyFavouriteMusic.Core/Features/MyFeatureExtractor.cs
using Larpx.PersonalTools.FindMyFavouriteMusic.Core.Interfaces;
using Larpx.PersonalTools.FindMyFavouriteMusic.Models.Results;

namespace Larpx.PersonalTools.FindMyFavouriteMusic.Core.Features;

/// <summary>
/// 自定义声学特征提取器
/// </summary>
public class MyFeatureExtractor : IAcousticFeatureExtractor
{
    /// <summary>特征维度</summary>
    public int FeatureDimension => 64;

    /// <summary>
    /// 从采样数据提取特征向量
    /// </summary>
    /// <param name="samples">单声道浮点采样</param>
    /// <param name="sampleRate">采样率</param>
    /// <returns>特征向量或失败结果</returns>
    public Result<float[]> Extract(float[] samples, int sampleRate)
    {
        var features = new float[FeatureDimension];
        // 实现特征提取逻辑
        return Result<float[]>.Success(features);
    }
}
```

### 7.2.2 注册到 DI

```csharp
// src/FindMyFavouriteMusic.GUI/App.axaml.cs
services.AddSingleton<IAcousticFeatureExtractor, MyFeatureExtractor>();
```

### 7.2.3 注意事项

- **维度一致性**：若改变了特征维度，需同步更新 `PredictionEngine` 的兼容性检查
- **数据迁移**：旧 BLOB 维度不兼容，需删除数据库中的 Songs 记录后重新扫描
- **画像重建**：更换提取器后必须重建用户画像

```csharp
// src/FindMyFavouriteMusic.Core/Audio/AudioFormatDetector.cs
[ ".ogg" ] = AudioFormat.Ogg,
```

#### Step 3 · 在 `AudioDecoder` 中实现支持

```csharp
// src/FindMyFavouriteMusic.Core/Audio/AudioDecoder.cs
public bool SupportsFormat(AudioFormat format) => format switch
{
    // ...
    AudioFormat.Ogg => OperatingSystem.IsWindows(), // 视解码器支持而定
    _ => false
};

private static WaveStream CreateReader(string filePath, AudioFormat format) => format switch
{
    // ...
    AudioFormat.Ogg when OperatingSystem.IsWindows() => new MediaFoundationReader(filePath),
    _ => throw new NotSupportedException($"不支持的格式: {format}")
};
```

#### Step 4 · 更新 `ScanOptions.SupportedExtensions`

```json
"Scan": {
  "SupportedExtensions": [".mp3", ".wav", ".flac", ".m4a", ".ogg"]
}
```

#### Step 5 · 更新文件选择对话框的扩展名筛选

修改 `PredictionView.axaml.cs` 中的 `FileTypeFilter`。

## 7.4 扩展：替换相似度算法

实现 `ISimilarityCalculator` 接口（如欧氏距离、马氏距离）。

### 7.4.1 实现接口

```csharp
// src/FindMyFavouriteMusic.Core/Prediction/EuclideanSimilarityCalculator.cs
using Larpx.PersonalTools.FindMyFavouriteMusic.Core.Interfaces;
using Larpx.PersonalTools.FindMyFavouriteMusic.Models.Results;

namespace Larpx.PersonalTools.FindMyFavouriteMusic.Core.Prediction;

public class EuclideanSimilarityCalculator : ISimilarityCalculator
{
    public Result<double> Calculate(float[] vectorA, float[] vectorB)
    {
        if (vectorA.Length != vectorB.Length)
            return Result<double>.Failure("维度不匹配");

        double sum = 0;
        for (var i = 0; i < vectorA.Length; i++)
        {
            var diff = vectorA[i] - vectorB[i];
            sum += diff * diff;
        }

        var distance = Math.Sqrt(sum);
        // 将距离映射到 [0, 1] 相似度（需根据实际场景调整）
        var similarity = 1.0 / (1.0 + distance);
        return Result<double>.Success(similarity);
    }
}
```

### 7.4.2 注册到 DI

```csharp
services.AddSingleton<ISimilarityCalculator, EuclideanSimilarityCalculator>();
```

### 7.4.3 注意事项

- 相似度取值范围若非 `[-1, 1]`，需同步修改 `PredictionEngine.MapToScore` 的映射公式
- 当前公式 `(similarity + 1) / 2 * 100` 假设输入为 `[-1, 1]`

## 7.5 ONNX 模型集成

### 7.5.1 什么是 VGGish

VGGish 是 Google 在 AudioSet（200 万视频、5800 类音频事件）上预训练的卷积神经网络，输入为 log-mel 频谱图，输出 128 维嵌入向量。它能捕获 MFCC 等手工特征难以表达的高层语义（乐器、人声、环境音等）。

### 7.5.2 获取 VGGish ONNX 模型

1. 从 [ONNX Model Zoo](https://github.com/onnx/models) 或 TensorFlow 官方 [VGGish 仓库](https://github.com/tensorflow/models/tree/master/research/audioset/vggish) 获取预训练模型
2. 若取得的是 TensorFlow checkpoint，需用 [tf2onnx](https://github.com/onnx/tensorflow-onnx) 转换为 ONNX 格式：

   ```bash
   python -m tf2onnx.convert --saved-model vggish_saved_model --output vggish.onnx
   ```

3. 将 `vggish.onnx` 放置到任意可读路径，例如 `D:/Models/vggish.onnx`

> 模型输入应为形状 `[batch, 1, 96, 64]` 的 `float32` 张量，输出为 `[batch, 128]`。本项目代码假设此格式。

### 7.5.3 配置模型路径

**方式 A：编辑配置文件**

修改 `appsettings.json`（或 `usersettings.json`）：

```json
{
  "OnnxModel": {
    "VggishModelPath": "D:/Models/vggish.onnx",
    "EnableDeepFeatures": true
  }
}
```

重启应用，`DeepFeatureExtractor` 构造函数会自动加载模型。

**方式 B：通过 UI 配置（推荐）**

1. 打开"设置"页
2. 在"ONNX 模型路径"输入框填入模型绝对路径
3. 勾选"启用深度特征"
4. 点击"保存 ONNX 配置"（写入 `usersettings.json`）
5. 点击"加载模型"（立即加载到内存，无需重启）
6. 模型加载成功后，"当前模型已加载"指示灯亮起

### 7.5.4 验证深度特征已启用

进入"预测"页，对一首歌曲进行预测：

- **状态**显示"深度增强模式" → 深度特征已生效
- **状态**显示"声学模式" → 模型未加载或待预测歌曲/画像缺少深度向量

> ⚠️ 启用深度特征后，**已扫描的歌曲不会自动补提深度向量**。如需为历史歌曲补提，需删除数据库中的 Songs 记录后重新扫描。

## 7.6 扩展：替换其他 ONNX 模型

若想用其他音频模型替换 VGGish（如 OpenL3）：

### 7.6.1 修改 `DeepFeatureExtractor` 常量

```csharp
// src/FindMyFavouriteMusic.Core/Features/DeepFeatureExtractor.cs
private const int VggishOutputDimension = 128;   // 改为新模型输出维度
private const double FrameDurationSeconds = 0.96; // 改为新模型输入时长
private const int MelBins = 64;                   // 改为新模型 mel 频带数
private const int SpectrogramTimeSteps = 96;      // 改为新模型时间步数
```

### 7.6.2 调整输入预处理

若新模型输入格式不同（如不需要 log-mel 而是直接波形），需重写 `ComputeLogMelSpectrogram` 方法。

### 7.6.3 更新维度相关代码

- `PredictionEngine` 中深度向量维度检查
- `UserProfile` 深度均值向量维度
- 数据库中已有深度向量需重新提取

## 7.9 扩展：性能优化建议

### 7.9.1 扫描性能

- 增大 `Scan.MaxConcurrentProcessing`（默认 2），但注意 CPU 核心数与内存限制
- 启用深度特征会显著增加单文件耗时（VGGish 推理为 CPU 密集型），初次扫描可先关闭，扫描完成后再补提
- 已扫描过的文件不会重复处理

### 7.9.2 预测性能

- 缩短音频时长（截取前 30s）
- 使用量化版 ONNX 模型（INT8 量化）
- 在支持 AVX2/AVX-512 的 CPU 上运行，OnnxRuntime 会自动利用

### 7.9.3 数据库性能

- SQLite 默认配置已足够个人音乐库使用
- 若歌曲数 > 10000，可考虑为 `FilePath` 添加索引（已有 UNIQUE 约束自动建索引）

# 06 · 配置说明

> 返回 [Wiki 首页](Home) · 上一章 [05-功能使用](05-功能使用) · 下一章 [07-扩展开发](07-扩展开发)

配置文件位于 `src/FindMyFavouriteMusic.GUI/appsettings.json`，运行时会被复制到输出目录。用户运行时修改的设置写入同目录的 `usersettings.json`。

---

## 6.1 配置优先级

应用启动时按以下顺序加载配置源（**后者覆盖前者同名键**）：

```mermaid
flowchart LR
    A["appsettings.json<br/>基础默认值"] --> B["usersettings.json<br/>用户运行时设置"]
    B --> C["环境变量<br/>FINDMYFAVOURITEMUSIC_ 前缀"]
    C --> D["最终生效配置"]
```

| 优先级 | 配置源 | 用途 |
|--------|--------|------|
| 1（最低） | `appsettings.json` | 基础默认值，随构建产物分发 |
| 2 | `usersettings.json` | 用户运行时设置，由设置页 UI 写入 |
| 3（最高） | 环境变量 | 敏感配置覆盖，前缀 `FINDMYFAVOURITEMUSIC_` |

---

## 6.2 完整配置示例

```json
{
  "FeatureExtraction": {
    "MfccCoefficientCount": 13,
    "MelFilterBankSize": 26,
    "FrameDurationSeconds": 0.025,
    "HopDurationSeconds": 0.010,
    "TargetSampleRate": 16000,
    "EnableNormalization": false
  },
  "Prediction": {
    "AcousticWeight": 0.4,
    "DeepWeight": 0.6,
    "AcousticOnlyWeight": 1.0
  },
  "OnnxModel": {
    "VggishModelPath": null,
    "EnableDeepFeatures": false
  },
  "Database": {
    "ConnectionString": "Data Source=findmyfavouritemusic.db"
  },
  "Scan": {
    "SupportedExtensions": [ ".mp3", ".wav", ".flac", ".m4a" ],
    "MaxConcurrentProcessing": 2
  }
}
```

---

## 6.3 FeatureExtraction（特征提取）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `MfccCoefficientCount` | int | 13 | MFCC 系数数量，典型值 13~20 |
| `MelFilterBankSize` | int | 26 | 梅尔滤波器组数量，通常为 MFCC 系数数的 2 倍 |
| `FrameDurationSeconds` | double | 0.025 | 帧长（秒），25ms 对应语音短时平稳假设 |
| `HopDurationSeconds` | double | 0.010 | 帧移（秒），10ms 对应 60% 重叠 |
| `TargetSampleRate` | int | 16000 | 目标采样率（Hz），VGGish 与多数声学模型使用 16kHz |
| `EnableNormalization` | bool | false | 是否对最终聚合向量做 Z-Score 归一化 |

> ⚠️ 修改此节后需重新扫描音乐库并重建画像，旧特征向量维度/尺度可能不兼容。

### 6.3.1 参数调优建议

| 场景 | 建议调整 |
|------|----------|
| 提升 MFCC 区分力 | `MfccCoefficientCount` 增至 20（维度变为 40+2+24=66） |
| 更精细的时间分辨率 | 减小 `HopDurationSeconds`（如 0.005），但会增加计算量 |
| 实验归一化效果 | `EnableNormalization: true`，但需同步重建所有特征与画像 |

---

## 6.4 Prediction（预测权重）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `AcousticWeight` | double | 0.4 | 声学+深度模式下的声学权重 |
| `DeepWeight` | double | 0.6 | 声学+深度模式下的深度权重 |
| `AcousticOnlyWeight` | double | 1.0 | 仅声学模式下的声学权重 |

> 通过设置页 UI 调整后会写入 `usersettings.json`，应用通过 `reloadOnChange: true` 监听变更。

### 6.4.1 权重约束

- `AcousticWeight + DeepWeight` 建议为 **1.0**
- `AcousticOnlyWeight` 建议为 **1.0**（避免缩放）
- 权重可为任意非负数，最终分数会被 `Math.Clamp` 截断到 [0, 100]

---

## 6.5 OnnxModel（ONNX 模型）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `VggishModelPath` | string? | null | VGGish ONNX 模型文件绝对路径 |
| `EnableDeepFeatures` | bool | false | 是否启用深度特征提取 |

### 6.5.1 配置示例

```json
{
  "OnnxModel": {
    "VggishModelPath": "D:/Models/vggish.onnx",
    "EnableDeepFeatures": true
  }
}
```

> 模型获取方式见 [07-扩展开发 · ONNX 模型集成](07-扩展开发#onnx-模型集成)。

## 6.6 Database（数据库）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `ConnectionString` | string | `Data Source=findmyfavouritemusic.db` | SQLite 连接字符串 |

### 6.6.1 自定义数据库位置

```json
{
  "Database": {
    "ConnectionString": "Data Source=D:/Data/fmm.db"
  }
}
```

## 6.7 Scan（扫描）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `SupportedExtensions` | string[] | `[".mp3", ".wav", ".flac", ".m4a"]` | 扫描时识别的音频扩展名 |
| `MaxConcurrentProcessing` | int | 2 | 最大并发解码+特征提取数 |

### 6.7.1 并发调优

- **增大** `MaxConcurrentProcessing`（如 4）：扫描更快，但 CPU 与内存压力增大
- **减小**（如 1）：扫描更慢，但资源占用更低，适合低配设备
- 启用深度特征时建议保持默认 2，VGGish 推理为 CPU 密集型

## 6.8 环境变量覆盖

环境变量层级用**双下划线** `__` 分隔，对应 JSON 的层级。

### 6.8.1 示例

```bash
# Windows PowerShell
$env:FINDMYFAVOURITEMUSIC_Database__ConnectionString = "Data Source=D:/Data/fmm.db"
$env:FINDMYFAVOURITEMUSIC_Prediction__AcousticWeight = "0.5"
$env:FINDMYFAVOURITEMUSIC_Prediction__DeepWeight = "0.5"

# Linux/macOS
export FINDMYFAVOURITEMUSIC_Database__ConnectionString="Data Source=/var/data/fmm.db"
export FINDMYFAVOURITEMUSIC_Prediction__AcousticWeight="0.5"
export FINDMYFAVOURITEMUSIC_Prediction__DeepWeight="0.5"
```

### 6.8.2 命名规则

| JSON 路径 | 环境变量名 |
|-----------|-----------|
| `Database.ConnectionString` | `FINDMYFAVOURITEMUSIC_Database__ConnectionString` |
| `Prediction.AcousticWeight` | `FINDMYFAVOURITEMUSIC_Prediction__AcousticWeight` |
| `OnnxModel.VggishModelPath` | `FINDMYFAVOURITEMUSIC_OnnxModel__VggishModelPath` |

> 前缀 `FINDMYFAVOURITEMUSIC_` 在 `App.axaml.cs` 中通过 `AddEnvironmentVariables("FINDMYFAVOURITEMUSIC_")` 注册。

## 6.9 配置不生效排查

若修改配置后未生效，请检查：

1. **文件位置**：运行时读取的是**输出目录**下的 `appsettings.json` / `usersettings.json`，而非源码目录
   - 输出目录：`src/FindMyFavouriteMusic.GUI/bin/Debug/net10.0/`
2. **热重载限制**：`reloadOnChange: true` 已开启，但部分配置（如 `FeatureExtraction`）只在服务构造时读取一次，需**重启应用**
3. **环境变量覆盖**：环境变量优先级最高，确认未误设 `FINDMYFAVOURITEMUSIC_*` 变量
4. **JSON 格式**：检查 JSON 语法是否正确（逗号、引号、括号匹配）

## 6.10 配置类与 Options 模式

所有配置通过 `IOptions<T>` 模式强类型注入：

```csharp
// 配置类示例
public class FeatureExtractionOptions
{
    public const string SectionName = "FeatureExtraction";

    public int MfccCoefficientCount { get; init; } = 13;
    public int MelFilterBankSize { get; init; } = 26;
    public double FrameDurationSeconds { get; init; } = 0.025;
    public double HopDurationSeconds { get; init; } = 0.010;
    public int TargetSampleRate { get; init; } = 16000;
    public bool EnableNormalization { get; init; } = false;
}

// 注册
services.Configure<FeatureExtractionOptions>(
    context.Configuration.GetSection(FeatureExtractionOptions.SectionName));

// 注入使用
public class AcousticFeatureExtractor
{
    private readonly FeatureExtractionOptions _options;
    public AcousticFeatureExtractor(IOptions<FeatureExtractionOptions> options) => _options = options.Value;
}
```
